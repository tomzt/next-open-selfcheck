# Internationalization (i18n)

Both apps use **[next-intl](https://next-intl-docs.vercel.app/)** for i18n — the standard for Next.js App Router.

---

## Supported Locales

| Code | Language | Status |
|---|---|---|
| `th` | ภาษาไทย (Thai) | ✅ Baseline |
| `en` | English | ✅ Baseline |

Thai and English are the baseline locales — both must always be complete. Community contributors can add any additional locale.

---

## File Structure

```
apps/kiosk/
  messages/
    th.json       ← Thai strings (baseline)
    en.json       ← English strings (baseline)
    [locale].json ← Add your locale here
  i18n.ts         ← next-intl config + locale list
  middleware.ts   ← Locale detection + routing
```

---

## Adding a New Locale

### 1. Create the locale file

```bash
cp apps/kiosk/messages/en.json apps/kiosk/messages/[locale].json
# e.g. cp apps/kiosk/messages/en.json apps/kiosk/messages/ja.json
```

### 2. Translate all strings

Translate every value in `[locale].json`. Keys must stay identical to `en.json`.

```json
{
  "welcome": {
    "cta": "タップして開始",
    "cta_sub": "または学生証をスキャン"
  }
}
```

### 3. Register the locale

Add your locale code to `apps/kiosk/i18n.ts`:

```typescript
export const locales = ['th', 'en', 'ja'] as const
```

### 4. Test

```bash
cd apps/kiosk
npm run dev
# Navigate to http://localhost:3000/ja
```

## Using Translations in Components

```typescript
import { useTranslations } from 'next-intl'

export default function WelcomeScreen() {
  const t = useTranslations('welcome')

  return (
    <button>{t('cta')}</button>
  )
}
```

### With variables

```typescript
const t = useTranslations('sip2')

// In th.json: "connecting_retry": "กำลังเชื่อมต่อ... (ครั้งที่ {current}/{total})"
t('connecting_retry', { current: 2, total: 3 })
// → "กำลังเชื่อมต่อ... (ครั้งที่ 2/3)"
```

## Runtime Language Switching

Patron can switch language at any time via the 🌐 button. Setting resets to admin default after session timeout.

```typescript
import { useRouter } from 'next/navigation'
import { useLocale } from 'next-intl'

export function LanguageButton() {
  const router = useRouter()
  const locale = useLocale()

  const switchLocale = (newLocale: string) => {
    router.replace(`/${newLocale}`)
  }
}
```

## Default Language

Set via environment variable — no code change needed:

```env
# apps/kiosk/.env
NEXT_PUBLIC_DEFAULT_LANGUAGE=th   # th | en | any registered locale
```

## Locale Strings Reference

See `apps/kiosk/messages/en.json` for the full list of translation keys.

Key namespaces:

| Namespace | Contents |
|---|---|
| `common` | Shared labels (loading, cancel, confirm...) |
| `welcome` | Welcome screen CTA text |
| `auth` | Login / logout messages |
| `menu` | Main menu labels |
| `transaction` | Borrow, Return, Status, Fines screens |
| `sip2` | Connection status + retry messages |
| `session` | Timeout warnings |
| `accessibility` | Font, theme, language button labels |
| `error` | Error messages |
